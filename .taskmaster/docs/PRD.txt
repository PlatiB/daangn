# Overview  
당근마켓 랜딩페이지 클론 프로젝트는 모던 프론트엔드 프레임워크 강의를 위한 실습 프로젝트입니다. React를 사용하여 당근마켓의 메인 랜딩페이지를 완전히 복제하는 것을 목표로 하며, 학습자들이 React의 핵심 개념과 모던 웹 개발 기술을 실습할 수 있도록 설계되었습니다.

이 프로젝트는 실제 서비스의 UI/UX를 따라 구현함으로써 실무에 가까운 개발 경험을 제공하며, 컴포넌트 기반 아키텍처, 상태 관리, 반응형 디자인, 그리고 모던 CSS 기법을 학습할 수 있습니다.

# Core Features  
## 1. 헤더 네비게이션
- 당근마켓 로고 구현
- "앱 다운로드" 버튼
- 반응형 헤더 디자인

## 2. 메인 히어로 섹션
- 동적 위치 기반 메시지 ("정자동에서 맛집 찾고 계신가요?")
- 지역 선택 드롭다운 컴포넌트
- 통합 검색 입력창
- 인기 검색어 자동완성 태그들

## 3. 카테고리 아이콘 그리드
- 7개 주요 카테고리 (중고거래, 알바, 부동산, 중고차, 동네업체, 동네생활, 모임)
- 아이콘과 텍스트가 결합된 카드형 레이아웃
- 호버 효과 및 클릭 인터랙션

## 4. 검색 태그 섹션
- 동적으로 생성되는 인기 검색어 태그
- 태그 클릭 시 검색창에 자동 입력 기능

## 5. 반응형 디자인
- 모바일, 태블릿, 데스크톱 대응
- CSS Grid 및 Flexbox 활용
- 미디어 쿼리를 통한 반응형 레이아웃

# User Experience  
## 사용자 페르소나
- **주요 사용자**: React 학습자 (초급-중급)

4. 카테고리 카드 호버/클릭 → 시각적 피드백
5. 인기 검색어 태그 클릭 → 검색창에 자동 입력

- 직관적인 사용자 인터페이스

# Technical Architecture  
## 시스템 컴포넌트
- **React 18**: 최신 React 기능 활용 (Hooks, Functional Components)
- **Parcel**: Zero-configuration 번들러 및 개발 서버
- **TypeScript**: 타입 안정성 확보
- **CSS Modules/Styled-components**: 컴포넌트 스타일링
- **React Router**: SPA 라우팅 및 네비게이션
- **HTTP 클라이언트**: fetch API 및 axios를 통한 API 통신
- **상태관리**: useState → Context API → Redux Toolkit → Jotai (단계별 진화)

## 컴포넌트 구조
```
App
├── Header
│   ├── Logo
│   └── DownloadButton
├── HeroSection
│   ├── LocationSelector
│   ├── SearchBar
│   └── PopularTags
├── CategoryGrid
│   └── CategoryCard (x7)
└── Footer (선택사항)
```

## 데이터 모델
- **지역 데이터**: 드롭다운용 지역 목록 (JSON)
- **카테고리 데이터**: 카테고리 정보 (아이콘, 텍스트, 링크)
- **검색어 데이터**: 인기 검색어 목록

## 상태 관리 전략 (단계별 학습)
- **Phase 1-3**: 컴포넌트 기본 구조 및 UI 구현
- **Phase 3.5**: Props와 State 기본 개념 확립
- **Phase 4-5**: useState를 활용한 로컬 상태 관리
- **Phase 6-7**: API 호출 및 라우팅을 통한 복잡한 상태 처리
- **Phase 8**: React Context API 도입 및 전역 상태 관리
- **Phase 10**: Redux Toolkit으로 전환 실습
- **Phase 11**: Jotai로 전환 실습 (선택사항)
- **비교 학습**: 각 상태관리 도구의 장단점 이해

## 스타일링 접근법
- CSS Grid/Flexbox 기반 레이아웃
- CSS Custom Properties (CSS 변수) 활용
- 모바일 퍼스트 반응형 디자인
- 부드러운 전환 효과 및 호버 애니메이션

### 🏗️ Main Project
- Parcel + React + TypeScript 프로젝트 생성
- 기본 폴더 구조 및 컴포넌트 파일 생성
- CSS Reset 및 기본 스타일 설정
- 당근마켓 브랜드 컬러 및 폰트 적용

### 📚 Concepts Setup
- 개념별 실습을 위한 폴더 구조 생성
- 각 concepts/ 하위 폴더별 독립적인 React 앱 설정
- playground/ 실험 환경 구축

### 📋 Design Reference (개발 참고용)
- **개발자 도구 Elements 탭**: 실제 렌더링된 HTML 구조 추출
- **Computed Styles**: 적용된 CSS 스타일 값들 수집
- **Assets 수집**: 필요한 이미지, 아이콘, 폰트 파일들
- `/reference/` 폴더에 정리하여 픽셀 퍼펙트 구현 참조

## Phase 2: 헤더 컴포넌트 구현 & 픽셀 퍼펙트 스타일링 ⭐ **외관 우선**
- **Reference 기반 구현**: `/reference/rendered-html/header.html` 구조 정확히 준수
- **원본 스타일 적용**: `/reference/computed-styles/header-styles.css` 값들 활용
- Logo, DownloadButton 컴포넌트 구현 (당근마켓과 동일한 외관)
- 헤더 레이아웃 및 스타일링 (픽셀 퍼펙트 구현)
- 반응형 헤더 구현 (원본과 동일한 브레이크포인트)
- **시스템 테마 대응**: prefers-color-scheme 활용한 다크/라이트 모드

## Phase 3: 메인 히어로 섹션 구현 & 외관 완성 ⭐ **외관 우선**
- **Reference 기반 구현**: `/reference/rendered-html/hero-section.html` 구조 정확히 준수
- **복합 검색창 구현**: 지역 선택 + 서비스 선택 + 검색 입력창 (원본과 동일)
- **애니메이션 텍스트 영역**: 고정폭 wrapper로 줄바꿈 방지 필수
- **정적 UI 완성**: 모든 시각적 요소를 먼저 완성 (기능은 나중에)
- **인기검색어 올바른 구현**: 작은 텍스트 링크 (큰 버튼 ❌)
- **모바일/데스크톱 반응형**: 원본과 동일한 레이아웃 변화

## Phase 3.5: 카테고리 그리드 & 전체 외관 완성 ⭐ **외관 우선**
- **Reference 기반 구현**: `/reference/rendered-html/category-grid.html` 구조 준수
- **7개 카테고리 완성**: 중고거래, 알바, 부동산, 중고차, 동네업체, 동네생활, 모임
- **아이콘 + 텍스트 구조**: 원본과 동일한 카드 레이아웃
- **반응형 그리드**: 모바일 1열 → 태블릿 2열 → 데스크톱 4열
- **임시 컨텐츠 제거**: 모든 테스트/더미 데이터 정리
- **전체 외관 점검**: 당근마켓과 최대한 유사한 완성도 확보

## Phase 4: Props와 State 기본 개념 & 컴포넌트 분리 🔧 **기능 시작**
- **컴포넌트 분리**: HeroSection, SearchForm, PopularTags, CategoryGrid 독립적 분리
- **Props 패턴**: 부모에서 자식으로 데이터 전달 (location, categories 등)
- **기본 State**: 간단한 상태 관리 (검색어 입력, 드롭다운 열림/닫힘)
- **mockData.js 활용**: 하드코딩 대신 데이터 파일 import 사용
- **이벤트 핸들링**: 기본적인 onClick, onChange 이벤트 처리
- **상태 끌어올리기**: 형제 컴포넌트 간 데이터 공유 패턴

## Phase 5: 동적 기능 구현 & 생명주기 관리 🔧 **기능 확장**
- **애니메이션 텍스트**: 동적으로 변경되는 키워드 구현 (줄바꿈 방지 유지)
- **Controlled Components**: 검색창 입력 상태 관리
- **useEffect Hook**: 컴포넌트 생명주기 이해 (애니메이션 타이머)
- **이벤트 핸들러**: 태그 클릭 시 검색창 연동 기능
- **조건부 렌더링**: 상태에 따른 UI 변화 (드롭다운, 호버 효과)
- **cleanup 함수**: 타이머 정리로 메모리 누수 방지

## Phase 6: API 호출 & 비동기 처리
- **API 호출**: fetch API 및 axios 활용
  - 지역 목록 데이터 API 호출
  - 인기 검색어 데이터 가져오기
  - 카테고리 정보 API 연동
- **비동기 상태 관리**: 
  - 로딩, 성공, 에러 상태 처리
  - async/await 패턴 구현
  - useEffect와 API 호출 연동
- **에러 핸들링**: API 실패 시 사용자 친화적 메시지 표시

## Phase 7: React Router 도입
- **React Router 설정**: 
  - BrowserRouter, Routes, Route 구성

- **네비게이션**: 
  - 카테고리 클릭 시 해당 페이지로 이동
  - 검색 시 결과 페이지로 라우팅
  - useNavigate, useParams Hook 활용
- **동적 라우팅**: 지역별, 카테고리별 URL 파라미터 처리

## Phase 8: Context API 상태관리 도입
- React Context API 설정

- Provider 패턴 구현

- **모바일 UX 최적화**: 터치 인터페이스, 스크롤 성능 개선

## Phase 10: Redux 상태관리 전환 실습 (선택사항) 🔧 **고급 기능**

```
│   │   ├── header.html    # 헤더 영역 HTML
│   │   ├── hero-section.html # 히어로 섹션 HTML
│   │   └── category-grid.html # 카테고리 그리드 HTML
│   ├── computed-styles/    # Computed 탭에서 추출한 스타일
│   │   ├── header-styles.css
│   │   ├── hero-styles.css
│   │   └── category-styles.css
│   ├── data/              # 표준 데이터 구조
│   │   └── mockData.js    # 카테고리, 태그, 위치 데이터
│   ├── design-tokens.txt   # 컬러, 폰트, 간격 등 정리
│   └── assets/             # 이미지, 아이콘 에셋
└── main/                   # 단일 React 프로젝트 (Git Checkpoint 방식)
    ├── src/
    │   ├── components/     # React 컴포넌트들
    │   ├── data/          # 프로젝트용 데이터 (reference/data 복사본)
    │   ├── styles/        # CSS 파일들
    │   └── App.js         # 메인 앱 컴포넌트
    ├── public/            # 정적 파일들
    ├── package.json       # 의존성 관리
    └── README.md          # 프로젝트 가이드
```

### 4. Git Checkpoint 시스템
기존 phase별 폴더 대신 **Git Checkpoint**를 사용하여 단계별 학습을 지원:

```bash
# 각 Phase 완료 시점의 Git 태그/브랜치
git checkout checkpoint-phase-1  # 프로젝트 초기 설정
git checkout checkpoint-phase-2  # 헤더 컴포넌트 완성
git checkout checkpoint-phase-3  # 히어로 섹션 기본 구조
git checkout checkpoint-phase-4  # Props & State 적용
git checkout checkpoint-phase-5  # 폼 입력 & useEffect
git checkout checkpoint-phase-6  # 카테고리 그리드 완성
git checkout checkpoint-phase-7  # API 호출 & 비동기 처리
git checkout checkpoint-phase-8  # React Router 적용
git checkout checkpoint-phase-9  # Context API 전환
git checkout checkpoint-phase-10 # 반응형 & 최적화 완성
```

### 5. 학습 패턴
1. **점진적 구현**: main/ 프로젝트에서 단계별 기능 추가
2. **개념 체험**: 구현 과정에서 자연스럽게 React 개념 습득
3. **체크포인트 활용**: 막힐 때 이전 안정 상태로 복구
4. **Git 워크플로우**: 실제 개발팀 협업 방식 체험

# Risks and Mitigations  
## 기술적 도전 과제

### ⭐ **외관 완성도 관련 리스크**
- **리스크**: 실습자가 원본과 다른 인기검색어 구현 (큰 버튼 형태)
- **해결방안**: 명확한 가이드와 Reference HTML 구조 필수 준수 강조

- **리스크**: 애니메이션 텍스트로 인한 레이아웃 변화 (줄바꿈)
- **해결방안**: 고정폭 wrapper와 white-space: nowrap 필수 적용

- **리스크**: 검색창을 단순한 input으로 구현하는 실수
- **해결방안**: 복합 구조(지역+서비스+검색) 필수 구현 가이드

- **리스크**: Reference 자료 무시하고 임의 구현
- **해결방안**: Phase별로 Reference 확인 단계 필수 포함

### 🔧 **교육적 도전 과제**
- **리스크**: React 초보자의 컴포넌트 구조 이해 부족
- **해결방안**: 외관 완성 후 점진적 컴포넌트 분리 접근

- **리스크**: 기능 구현을 우선시하여 외관 완성도 저하
- **해결방안**: Phase 1-3.5는 순수 외관에만 집중, 기능은 Phase 4부터

- **리스크**: 시스템 테마 미대응으로 사용자 경험 저하
- **해결방안**: prefers-color-scheme을 활용한 자동 테마 적용 필수

### 🎨 **디자인 관련 리스크**
- **리스크**: 당근마켓 디자인의 정확한 복제 어려움
- **해결방안**: 상세한 Reference 자료와 픽셀 퍼펙트 구현 가이드

- **리스크**: 반응형 디자인 구현의 복잡성
- **해결방안**: Reference 기반 모바일/데스크톱 정확한 브레이크포인트 적용

- **리스크**: 임시 테스트 컨텐츠로 인한 완성도 저하
- **해결방안**: Phase 3.5에서 모든 더미 데이터 제거 단계 필수

### ⚖️ **법적/윤리적 리스크**
- **리스크**: 당근마켓 디자인 참고에 따른 저작권 이슈
- **해결방안**: 
  - 교육 목적으로만 사용하고 상업적 배포 금지
  - 원본 HTML/CSS는 참고용으로만 활용
  - 최종 결과물은 독립적인 React 컴포넌트로 구현
  - 당근마켓 로고/이미지는 대체 이미지 사용

## MVP 정의 (외관 우선 교육용)
- **흥미 유발 MVP**: Phase 1-3.5 완료 시점 (완벽한 외관 + 시스템 테마 대응)
  - 당근마켓과 구별하기 어려운 수준의 시각적 완성도
  - 실습자 즉시 만족과 동기부여 극대화
- **기능 학습 MVP**: Phase 1-5 완료 시점 (외관 + 기본 React 개념)
  - Props, State, useEffect 마스터
  - 동적 애니메이션과 기본 상호작용
- **실무 준비 MVP**: Phase 1-9 완료 시점 (성능 최적화까지 완성)
  - 포트폴리오 수준의 완성도
  - 실제 프로젝트 배포 가능 수준
- **핵심 가치**: 즉시 시각적 만족 → 학습 동기 부여 → React 개념 습득
- **확장 가능**: 추후 실제 백엔드 연동 및 고급 기능 확장 가능

## 자원 제약 고려 (교육 관점)

  - 개념 학습 (concepts/) + 프로젝트 적용 (main/) 이중 구조로 효율적 학습

- **학습자 자율성**:
