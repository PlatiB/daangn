실습자들이 AI 도구(Cursor, Claude, ChatGPT 등)를 사용할 때 일관된 결과를 얻을 수 있도록 하는 프롬프트 템플릿 모음입니다.

## 📋 **기본 컨텍스트 설정 프롬프트**

### **실습 시작 시 사용할 기본 프롬프트**
```
당근마켓 메인 페이지를 React로 클론 코딩하는 실습을 진행하고 있습니다.

[프로젝트 요구사항]
- React 함수형 컴포넌트 사용
- 4개 주요 섹션 구현: HeroSection, CategoryGrid, PopularTags, PopularLocations
- 반응형 디자인 적용 (모바일/태블릿/데스크톱)
- 표준화된 데이터 구조 사용

[필수 구현 컴포넌트]
1. HeroSection: 메인 타이틀 + 동적 텍스트 애니메이션 + 검색창
2. CategoryGrid: 7개 카테고리 반응형 그리드 (중고거래, 알바, 부동산, 중고차, 동네업체, 동네생활, 모임)
3. PopularTags: 17개 인기 검색어 가로 스크롤
4. PopularLocations: 20개 인기 지역 flex wrap

[데이터 표준]
- src/data/mockData.js 파일의 데이터 사용 필수
- 하드코딩 금지, 표준 색상 팔레트 사용
- 당근마켓 브랜드 컬러: #FF6F0F (primary), #FF9F30 (secondary)

[금지사항]
- HTML 복사 붙여넣기
- 모든 코드를 App.js에 작성
- 인라인 스타일 남발
- 컴포넌트 미분리

위 요구사항을 바탕으로 코딩을 도와주세요.
```

## 💡 **AI 도구별 활용 팁**

### **Cursor에서 사용 시**
- `Ctrl+K`로 인라인 편집 모드에서 위 프롬프트 사용
- `@파일명`으로 특정 파일 컨텍스트 포함
- `#터미널`로 에러 로그와 함께 질문

### **Claude/ChatGPT에서 사용 시**
- 프로젝트 컨텍스트 설정 후 세부 질문 진행
- 코드 블록으로 현재 상황 명확히 제시
- 단계별 구현 요청으로 학습 효과 극대화

### **효과적인 질문 패턴**
1. **상황 설명** → **구체적 요구사항** → **제약 조건** → **기대 결과**
2. 에러 발생 시: **에러 메시지** + **관련 코드** + **시도한 해결책**
3. 리팩토링 시: **현재 코드** + **문제점** + **개선 목표**
