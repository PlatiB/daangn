# 👨‍🏫 강사용 AI 기반 실습 운영 가이드

## 📋 개요

이 가이드는 **AI 컨텍스트와 Cursor Rules를 활용한 당근마켓 클론 실습**을 효과적으로 운영하는 방법을 제시합니다.

---

## 🎯 **AI 기반 표준화 시스템의 장점**

### **✅ 강사 관점에서의 이점**
- **일관된 교육 품질**: 모든 실습자가 동일한 가이드라인을 받음
- **반복 질문 감소**: AI가 표준 답변을 제공
- **개별 지도 효율성**: 공통 문제는 AI가, 특수 케이스는 강사가 담당
- **실시간 피드백**: 실습자가 즉시 도움을 받을 수 있음

### **✅ 실습자 관점에서의 이점**
- **24/7 멘토링**: 언제든지 AI에게 질문 가능
- **개인별 맞춤 속도**: 각자의 속도에 맞춰 학습
- **부담 없는 질문**: 강사에게 묻기 부끄러운 기초 질문도 자유롭게
- **일관된 코드 품질**: 표준화된 패턴으로 구현

```
✅ 언제든지 AI에게 질문할 수 있습니다

- 막히면 AI에게 먼저 질문, 해결 안 되면 강사에게"
```

## 📊 **단계별 실습 운영**

### **1단계: 프로젝트 셋업 (20분)**

#### **강사 역할:**
- 전체적인 프로젝트 구조 설명
- AI 활용 방법 시연
- 공통 환경 설정 문제 해결

#### **AI가 담당하는 부분:**
- React 프로젝트 생성 가이드
- 폴더 구조 생성
- 기본 설정 파일 작성

#### **강사 모니터링 포인트:**
```
✅ 체크할 항목:
- 모든 실습자의 프로젝트가 정상 실행되는가?
- .cursor/rules 파일이 제대로 적용되었는가?
- AI 도구가 정상 작동하는가?
```

### **2단계: HeroSection 구현 (45분)**

#### **강사 역할:**
- 컴포넌트 설계 원칙 설명
- useState, useEffect 개념 리뷰
- 공통적으로 발생하는 고급 문제 해결

#### **AI가 담당하는 부분:**
- 기본 컴포넌트 구조 생성
- 애니메이션 로직 구현
- CSS 스타일링 가이드
- 기본적인 디버깅

#### **강사 개입이 필요한 상황:**
```
🚨 강사가 나서야 할 때:
- AI 답변이 일관되지 않을 때
- 실습자가 AI 답변을 이해하지 못할 때
- 창의적 개선 방향을 제시할 때
- 실무 관점의 조언이 필요할 때
```

### **3단계: CategoryGrid 구현 (60분)**

#### **핵심 학습 포인트:**
- CSS Grid vs Flexbox
- 반응형 디자인 패턴
- 컴포넌트 재사용성

#### **강사의 부가가치:**
```
AI가 제공하지 못하는 강사만의 가치:
🎯 실무 경험 공유
- "실제 프로젝트에서는 이런 패턴을 사용해요"
- "이 방법이 유지보수하기 더 좋은 이유는..."

🎯 창의적 개선 제안
- "기본 구현에서 한 단계 더 나아가려면..."
- "이런 추가 기능을 구현해보는 것도 좋겠네요"

🎯 팀워크 및 협업 관점
- "팀에서 일할 때는 이런 점을 고려해야 해요"
- "코드 리뷰에서 주로 지적받는 부분들..."
```

### **4단계: 나머지 컴포넌트 구현 (40분)**

#### **자기주도 학습 시간:**
- 실습자들이 AI를 활용해 독립적으로 진행
- 강사는 순회하며 개별 지도
- 고급 질문이나 창의적 개선에 집중

## 🔧 **문제 상황별 대응 가이드**

### **AI 답변이 일관되지 않을 때**
```
상황: 실습자들이 서로 다른 AI 답변을 받아서 혼란

대응 방법:
1. 표준 프롬프트 템플릿 재안내
2. 핵심 요구사항 재확인
3. 강사가 직접 올바른 방향 제시
4. 공통 질문은 전체 공유
```

### **실습자가 AI에 과도하게 의존할 때**
```
상황: AI가 모든 코드를 작성해주기만 바라는 경우

대응 방법:
1. "AI는 도우미, 이해가 목표"임을 강조
2. 코드 한 줄씩 설명하도록 요청
3. 변형 문제 제시 ("이 부분을 다르게 구현한다면?")
4. 페어 프로그래밍으로 전환
```

### **AI가 잘못된 답변을 제공할 때**
```
상황: AI가 실습 목표와 다른 방향으로 안내

대응 방법:
1. 즉시 수정된 방향 제시
2. 왜 그 방법이 더 좋은지 설명
3. 해당 케이스를 전체에게 공유
4. 필요시 Cursor Rule 업데이트
```
